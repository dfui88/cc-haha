1整体架构

Claude Code Haha 是一个基于 Anthropic Claude Code CLI 的桌面端 AI 聊天客户端，采用 Tauri 壳 + React 前端 + Bun 后端的三层架构。

四层结构

┌─────────────────────────────────────────────────────────────┐ │ Tauri 桌面壳 (Rust) │ │ 窗口管理 / 原生对话框 / 自动更新 / 进程生命周期 │ │ 终端伪终端 / 文件对话框 / 侧边进程(Sidecar)管理 │ ├─────────────────────────────────────────────────────────────┤ │ React 前端 (desktop/src/) │ │ TSX + Tailwind CSS 4 + Zustand 5 状态管理 │ │ Vite 6 构建 / 多标签页管理 / 流式渲染 │ ├─────────────────────────────────────────────────────────────┤ │ Bun 后端 (src/server/) │ │ Bun.serve() 原生 HTTP + WebSocket │ │ REST API (20+ 端点) + WS 实时通信 │ │ 管理 Claude CLI 子进程 │ ├─────────────────────────────────────────────────────────────┤ │ Claude CLI (AI 引擎) │ │ 实际对话处理 / 工具调用 / 流式输出 │ │ stream-json + vscode-jsonrpc 通信 │ └─────────────────────────────────────────────────────────────┘

路由设计

无传统路由库（无 React Router）。采用 Zustand 的 tabStore 管理多标签页：

tabStore.activeTabId → ContentRouter 组件
  ├── session-xxx → <ChatPage />        ← 聊天会话
  ├── settings    → <SettingsPage />     ← 设置
  ├── scheduled   → <ScheduledTasks />   ← 定时任务
  └── terminal    → <TerminalPage />     ← 终端

每个标签页独立持久化到 localStorage，支持关闭恢复和标题自动生成。

4前后端通信方式

采用双通道通信架构：REST API + WebSocket

REST API

前端 api.get('/api/sessions')
  → fetch('http://127.0.0.1:3456/api/sessions', { signal: AbortSignal.timeout(30000) })
  → Bun.serve() → router.ts → API handler
  ← JSON Response
参数值
默认端口3456
超时30 秒
认证localhost 免认证
CORS自动附加

WebSocket 实时消息

wsManager.connect(sessionId)
  → WebSocket('ws://127.0.0.1:3456/ws/{sessionId}')
能力说明
自动重连指数退避 1s ~ 30s
消息队列最多缓存 500 条
心跳每 30 秒 ping
连接隔离每个 sessionId 独立连接

5Tauri 桌面壳集成

Tauri 主进程 (Rust)
  │
  ├── beforeBuildCommand: node scripts/build-before.mjs
  │   ├── tsc -b && vite build          ← 前端构建
  │   └── bun run build:sidecars         ← 后端 sidecar 构建
  │
  ├── 加载前端:
  │   ├── 开发模式: http://localhost:1420 (Vite HMR)
  │   └── 生产模式: ../dist (静态文件)
  │
  ├── Rust Commands:
  │   ├── get_server_url()    → 返回后端地址
  │   ├── terminal_spawn()    → 启动伪终端
  │   ├── terminal_write()    → 写入终端
  │   ├── terminal_resize()   → 调整大小
  │   └── terminal_kill()     → 关闭终端
  │
  ├── Sidecar:
  │   ├── externalBin: ["binaries/claude-sidecar"]
  │   └── Bun.serve() 作为独立进程运行
  │
  └── 前端启动流程:
      1. 检测 Tauri 环境
      2. invoke('get_server_url')
      3. 设置 api baseUrl
      4. 轮询 /health 等待就绪（30 次 × 250ms）

6核心流程

聊天消息流程

用户输入 → ChatInput → WebSocket → Bun 服务端 → CLI 子进程 → AI 推理 
用户输入 → ChatInput.tsx
  → chatStore.sendMessage()
  → WebSocket { type: 'user_message' }
  → ws/handler.ts → handleUserMessage()
  → conversationService.sendMessage() 写入 CLI stdin
  → Claude CLI 推理 → stdout stream-json
  → conversationService.onOutput() 解析
  → WebSocket 推送: content_start → content_delta
    → tool_use_complete → tool_result → message_complete
  → chatStore.handleServerMessage()
  → MessageList.tsx 流式渲染

工具调用流程

AI 决定 → tool_use_complete → ToolCallBlock → CLI 执行 → tool_result → 送回 AI 
AI 决定调用工具
  → 服务端发送 tool_use_complete
  → 前端渲染 ToolCallBlock ("运行中...")
  → CLI 执行工具 (Bash/文件/搜索等 55+ 工具)
  → 服务端发送 tool_result
  → 前端 ToolResultBlock 渲染结果
  → 结果送回 AI 继续生成

AI Provider 代理流程

桌面端请求 (Anthropic Messages API 格式)
  → POST /api/proxy 或 /v1/messages
  → handleProxyRequest()
    ├── anthropic 格式 → 直接透传
    ├── openai_chat → anthropicToOpenaiChat() 转换
    └── openai_responses → anthropicToOpenaiResponses() 转换
  → 上游 AI API (Anthropic / OpenAI 兼容 / AWS Bedrock)
  → 响应转回 Anthropic 格式 → 返回前端

架构特点

  1. Sidecar 架构 — Bun 服务器作为独立进程运行，Tauri 管理其生命周期
  2. 双通道通信 — REST API 用于数据查询，WebSocket 用于实时消息推送
  3. Provider 适配层 — 支持任意 OpenAI 兼容 API，协议自动转换
  4. 流式渲染 — AI 响应通过 WebSocket 实时推送，前端逐块渲染
  5. 多标签页 — 无传统路由，通过 Zustand 管理标签页，支持持久化恢复
  6. 状态隔离 — 每个会话独立状态，23 个 Store 按职责拆分
  7. 超时保护 — 前端 30 分钟响应超时，WebSocket 心跳保持连接
  8. CLI 子进程管理 — 自动启动/停止/重启 Claude CLI 进程
