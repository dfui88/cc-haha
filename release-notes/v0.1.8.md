## 七、Bug 修复

### 7.1 工作目录无效错误（WORKDIR_INVALID）

**根因：** `sessionService.ts:634` 在未传 workDir 时默认使用 `os.homedir()`，在 Administrator 账户下路径格式错误导致 "工作目录无效或不存在"。

**修复：**

**路径：** `src/server/services/sessionService.ts`

| 修改 | 位置 | 说明 |
|------|------|------|
| `createSession` 移除 os.homedir() 兜底 | 第 668-683 行 | 无 workDir 时不设默认路径，在 projects 根目录创建 session |
| 新增 `WORKDIR_MISSING` 错误码 | `conversationService.ts` | 明确提示用户选择项目文件夹 |
| `findSessionFile` 增加根目录扫描 | 第 524-532 行 | 也检查 projects 根目录的 `.jsonl` 文件 |
| `discoverSessionFiles` 增加根目录 `.jsonl` 扫描 | 第 410-420 行 | 扫描根目录中的直接 `.jsonl` 文件 |

**路径：** `src/server/ws/handler.ts`

- 移除 `import * as os from 'node:os'`（第 12 行）
- `resolveSessionWorkDir` 不再回退到 `os.homedir()`，无 workDir 时返回空字符串

**路径：** `desktop/src/i18n/locales/zh.ts`、`desktop/src/i18n/locales/en.ts`

- 添加 `error.WORKDIR_MISSING` 翻译："请先为此会话选择一个项目文件夹。"

### 7.2 会话重命名不生效

**根因：** `handleFinishRename` 先调 API 再更新本地 store。API 失败时 catch 吃掉异常，store 不更新，但输入框正常关闭——表现：名字没变、输入框没了。

**修复（乐观更新）：**

**路径：** `desktop/src/components/layout/Sidebar.tsx`

| 修改 | 位置 | 说明 |
|------|------|------|
| 先更新本地 store，再调 API | 第 101-103 行 | 先调 `updateSessionTitle()` + `updateTabTitle()` 立即更新 UI |
| API 失败不影响本地 | 第 106-111 行 | 服务器同步失败时本地已更新，下次 fetchSessions 自动协调 |

### 7.3 会话重命名卡住

**根因：** `handleFinishRename` 没有 try/catch，`renameSession` 抛异常时 `setRenamingId(null)` 不执行，输入框一直显示。

**修复：** `desktop/src/components/layout/Sidebar.tsx` — 添加 try/catch，确保 `setRenamingId(null)` 始终执行。

### 7.4 findSessionFile 找不到根目录文件

**根因：** 无 workDir 的 session 存储在 projects 根目录（`<projectsDir>/<uuid>.jsonl`），但 `findSessionFile` 只搜索子目录。

**修复：** `src/server/services/sessionService.ts` — `findSessionFile()` 和 `discoverSessionFiles()` 增加根目录 `.jsonl` 检查。

### 7.5 Sidecar 时间戳不更新

**根因：** 构建脚本只更新了 MSI 产物的时间戳，没更新 `claude-sidecar.exe`。

**修复：** `desktop/scripts/build-windows-x64.ps1` — 在 tauri build 前同步 sidecar 时间戳。

### 7.6 删除会话条目和标签不消失

**根因：** `deleteSession` 先调 API 再更新本地 store，API 失败时（404 找不到文件）store 不更新、`closeTab` 和 `disconnectSession` 都不执行，只有对话框关闭。

**修复（乐观删除）：** `desktop/src/components/layout/Sidebar.tsx` — 先清理所有本地状态（关闭标签、断开连接、从列表移除、关闭对话框），再异步同步服务器。

### 7.7 切换项目文件夹出现 "New Session"

**根因：** `ChatInput.tsx:715` 的 DirectoryPicker 切换文件夹时调用 `createSession(newWorkDir)` 未传标题参数，新 session 的乐观标题为默认 "New Session"。同时旧 session 删除可能失败，导致侧边栏残留旧会话并多出一个 "New Session"。

**修复：** `desktop/src/components/chat/ChatInput.tsx` — 传入当前标签标题 `currentTab.title`，新 session 保持 "会话N" 编号，并同步 persist 到服务器。

**效果：** 选择项目文件夹时侧边栏不再出现重复会话条目。

## 八、AI 模型中文回复

桌面端语言设置为中文时，AI 模型的回复自动使用中文，无需用户每次手动提示。

### 8.1 前端 locale 同步到服务器

**路径：** `desktop/src/stores/settingsStore.ts`

|------|------|
| `setLocale()` 改为 async | 变更 locale 时同步调用 `settingsApi.updateUser({ locale })` 持久化到服务器 |
| `fetchAll()` 初始化同步 | 应用启动时自动将当前 locale 写入 `~/.claude/settings.json` |

### 8.2 服务端 locale 读取

**路径：** `src/server/ws/handler.ts`

| 修改 | 位置 | 说明 |
|------|------|------|
| `getRuntimeSettings()` 读取 `userSettings.locale` | 第 1090、1135 行 | 从 `settingsService.getUserSettings()` 获取 locale |
| 通过 `SessionStartOptions` 传递 | 返回值 | locale 随 model、effort 等一起传入会话配置 |

### 8.3 CLI 注入语言指令

**路径：** `src/server/services/conversationService.ts`

| 修改 | 位置 | 说明 |
|------|------|------|
| `SessionStartOptions` 新增 `locale` | 第 52 行 | `locale?: string` |
| `getRuntimeArgs()` 添加 `--append-system-prompt` | 第 521-523 行 | `locale === 'zh'` 时注入 `"请用中文回复。你是一位AI助手，请始终用中文回答用户的问题。"` |

### 数据流

```
桌面端设置 locale=zh
  → settingsApi.updateUser({ locale: 'zh' })
    → 写入 ~/.claude/settings.json
      → getRuntimeSettings() 读取 locale
        → startSession() 带上 locale
          → CLI 收到 --append-system-prompt "请用中文回复..."
            → buildEffectiveSystemPrompt() 追加到系统提示词末尾
              → AI 模型用中文回复
```

**效果：** 中文界面下 AI 模型自动使用中文回复；切换到英文界面后 AI 恢复英文回复（无 `--append-system-prompt` 注入）。
