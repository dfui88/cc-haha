# 需求头脑风暴

## 用途
在编码之前，对模糊的想法进行结构化头脑风暴，拆解为可执行的任务。

## 工作流程

1. **需求澄清**
   - 明确要做什么
   - 目标用户是谁
   - 核心价值是什么

2. **功能拆解**
   - 拆解为用户故事
   - 定义 MVP 范围
   - 标记优先级 (P0/P1/P2)

3. **技术评估**
   - 识别技术风险
   - 评估实现难度
   - 选择技术方案

4. **输出产物**
   - 任务列表
   - 技术方案要点
   - 验收标准
