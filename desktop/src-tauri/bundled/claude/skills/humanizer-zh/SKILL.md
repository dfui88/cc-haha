# 中文人性化润色技能 (Humanizer-ZH)

让 AI 生成的中文摆脱"机翻味"，读起来像真人写的。

## 常见 AI 中文问题

| 问题 | AI 风格 | 人类风格 |
|------|---------|---------|
| 过长定语 | 这是一个经过精心设计的、能够有效提升用户体验的解决方案 | 这个方案经过精心设计，能有效提升用户体验 |
| 生硬连接词 | 此外，值得注意的是，与此同时 | 另外，不过话说回来，同时 |
| 万能动词 | 进行、给予、提供 | 做、给、提供（更具体） |
| 过度正式 | 请知悉；敬请期待；毋庸置疑 | 跟你说一下；等着看吧；毫无疑问 |
| 啰嗦铺垫 | 在当今数字化浪潮的背景下 | （直接说重点） |
| 万能句式 | 值得注意的是……不可或缺的一部分……值得关注的是 | （直白表达） |

## 润色原则

### 1. 缩短句子

将长句拆成短句。中文天然适合短句。

### 2. 减少"被"字

中文被动语态用得少，主动语态更自然。

### 3. 去掉冗余修饰

检查每个形容词和副词——删掉后意思不变就删。

### 4. 口语化适度的平衡

根据不同场景控制口语化程度：
- 正式文档：稍做松弛，保留专业性
- 内部沟通：接近口语
- 技术文档：清晰直接，不用花哨

### 5. 使用自然过渡

口语化的连词：
- 不过、话说回来、对了、说到这个
- 所以呢、总之吧、说白了

## 使用场景

- 技术文档中的中文部分
- 内部沟通消息
- 用户界面中文文案
- 产品说明和博客文章

## 提示词示例

```
请润色以下中文文本，让它更像真人写的：
[粘贴文本]

要求：缩短长句，减少"进行/给予"类动词，不用"值得注意的是"这类AI句式。
```

## 注意事项

- 不要过度口语化导致失去专业性
- 保留技术术语不变
- 不同场景风格不同：文档 > 消息 > 界面文案
